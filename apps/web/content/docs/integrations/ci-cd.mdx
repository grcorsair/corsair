## Overview

Corsair runs in any CI/CD pipeline that supports Bun. Use it for continuous compliance verification — validate that compliance evidence is ingested, mapped, and signed on every release cycle.

## GitHub Actions

```yaml
name: Corsair Compliance Verification
on:
  schedule:
    - cron: '0 6 * * 1'  # Weekly Monday 6am UTC
  workflow_dispatch:

jobs:
  corsair:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: latest

      - name: Install Corsair
        run: |
          git clone https://github.com/Arudjreis/corsair.git /tmp/corsair
          cd /tmp/corsair && bun install

      - name: Ingest and Sign
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          cd /tmp/corsair
          bun corsair.ts ingest \
            --input ${{ github.workspace }}/compliance/soc2-report.pdf \
            --format oscal \
            --output /tmp/evidence.json

      - uses: actions/upload-artifact@v4
        with:
          name: corsair-evidence
          path: /tmp/evidence.json
```

## GitLab CI

```yaml
corsair:
  image: oven/bun:latest
  stage: test
  only:
    - schedules
  script:
    - git clone https://github.com/Arudjreis/corsair.git /tmp/corsair
    - cd /tmp/corsair && bun install
    - bun corsair.ts ingest --input compliance/soc2-report.pdf --format oscal --output evidence.json
  artifacts:
    paths:
      - evidence.json
    expire_in: 90 days
```

## Jenkins

```groovy
pipeline {
    agent any
    triggers { cron('H 6 * * 1') }
    stages {
        stage('Corsair Compliance Verification') {
            steps {
                sh '''
                    bun corsair.ts ingest \
                        --input compliance/soc2-report.pdf \
                        --output evidence/corsair-${BUILD_ID}.json \
                        --quiet
                '''
            }
        }
    }
    post {
        always {
            archiveArtifacts artifacts: 'evidence/*.json'
        }
    }
}
```

## Best Practices

1. **Schedule weekly** — Run ingestion and verification on a recurring schedule, not just on deploys
2. **Archive evidence** — Store OSCAL/JSONL artifacts for audit trail
3. **Alert on failures** — Send Slack/email notifications when CPOE assurance level drops
4. **Verify CPOEs** — Use `bun corsair.ts verify` to validate existing CPOEs in your pipeline
5. **Track assurance levels** — Monitor L0-L4 progression across release cycles
