## Overview

Corsair generates [OSCAL](https://pages.nist.gov/OSCAL/) (Open Security Controls Assessment Language) Assessment Results — the NIST standard for machine-readable security assessment data.

## Output Formats

```bash
# OSCAL JSON (machine-readable, NIST SP 800-53A compliant)
bun corsair.ts --target demo --service cognito --format oscal

# HTML report (self-contained, no external dependencies)
bun corsair.ts --target demo --service cognito --format html

# Markdown report
bun corsair.ts --target demo --service cognito --format md
```

## OSCAL Mapping

| Corsair Concept | OSCAL Element |
|----------------|---------------|
| ISC Criteria | Findings (satisfied/not-satisfied) |
| MARK Drift Findings | Observations (TEST method) |
| RAID Results | Risks (open/closed) |
| CHART Frameworks | Reviewed Controls (control-selections) |

## Report Sections

HTML and Markdown reports include:

- **Executive Summary** — Mission overview, ISC satisfaction rate, risk score
- **Framework Coverage** — Which frameworks mapped, control counts
- **Finding Details** — Per-finding severity, drift values, remediation context
- **Evidence Chain** — Hash chain verification, record count, integrity status
- **ISC Table** — Full criteria list with SATISFIED/FAILED status

## Integration with GRC Platforms

OSCAL JSON output can be ingested by:

- **ServiceNow** GRC module (via REST API import)
- **Archer** IRM (via data feed)
- **Jira** (JSONL evidence → Jira issues via webhook)
- **Drata/Vanta** (planned Corsair integration)

The OSCAL format ensures Corsair evidence is portable across any platform that supports the NIST standard.
