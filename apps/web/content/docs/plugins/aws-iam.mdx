## Overview

The AWS IAM plugin tests identity and access management hygiene across password policies, access key lifecycle, MFA enforcement, and privilege boundaries.

## Attack Vectors

### Password Policy Weakness (`password-policy-weakness`)

**Severity:** HIGH
**MITRE ATT&CK:** T1110 (Brute Force)

Tests the account-level password policy for minimum length, complexity requirements, rotation period, and password reuse prevention.

### Stale Access Keys (`stale-access-keys`)

**Severity:** HIGH
**MITRE ATT&CK:** T1078 (Valid Accounts)

Identifies access keys that haven't been rotated within the recommended 90-day window. Stale keys increase the blast radius of credential compromise.

### MFA Not Enforced (`mfa-not-enforced`)

**Severity:** CRITICAL
**MITRE ATT&CK:** T1556 (Modify Authentication Process)

Checks whether MFA is enforced for all IAM users, especially those with console access. Detects users without any MFA device configured.

### Privilege Escalation Path (`privilege-escalation-path`)

**Severity:** CRITICAL
**MITRE ATT&CK:** T1548 (Abuse Elevation Control Mechanism)

Analyzes IAM policies for potential privilege escalation paths â€” users who can modify their own permissions, assume high-privilege roles, or create new access keys for other users.

## Usage

```bash
# Demo mode
bun corsair.ts --target demo --service iam

# Live mode (scans entire account)
bun corsair.ts --target account --service iam
```
