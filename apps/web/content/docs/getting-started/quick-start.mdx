## Prerequisites

- [Bun](https://bun.sh/) runtime (v1.0+)

## Installation

```bash
# Clone the repository
git clone https://github.com/Arudjreis/corsair.git
cd corsair

# Install dependencies
bun install
```

## Verify an Existing CPOE (No API Key Needed)

The fastest way to see Corsair in action â€” verify the example CPOE:

```bash
bun run bin/corsair-verify.ts examples/example-cpoe.jwt
```

This verifies the Ed25519 signature, checks expiry, and displays the provenance and control results.

## Sign Tool Output into a CPOE

Corsair takes structured output from security tools and signs it into a cryptographically verifiable CPOE:

```bash
# Sign Prowler scan results
corsair sign --file prowler-results.json --format prowler

# Sign InSpec compliance results
corsair sign --file inspec-results.json --format inspec

# Sign Trivy vulnerability scan
corsair sign --file trivy-report.json --format trivy
```

## Compare CPOEs Over Time

Track how your compliance posture changes between assessments:

```bash
# Compare two CPOEs (like git diff for compliance)
corsair diff --current cpoe-v2.jwt --previous cpoe-v1.jwt
```

## Generate Signing Keys

```bash
# Generate Ed25519 keypair for CPOE signing
corsair keygen --output ./keys
```

## CLI Commands

| Command | Description |
|---------|-------------|
| `corsair sign` | Sign security tool output into a verifiable CPOE |
| `corsair verify` | Verify a CPOE's Ed25519 signature and check validity |
| `corsair diff` | Compare two CPOEs and show control-level changes |
| `corsair log` | List signed CPOEs (local SCITT log) |
| `corsair signal` | FLAGSHIP real-time compliance notifications |
| `corsair keygen` | Generate Ed25519 signing keypair |
| `corsair help` | Show available commands |
